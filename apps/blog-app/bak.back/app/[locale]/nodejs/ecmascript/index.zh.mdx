# ECMAScript简介

1993 年第一个带有用户界面的 Web 浏览器Mosaic, 于 1994 年发布，Mosaic 的主要开发人员创立了一家名为 Netscape 的公司，并发布了一款更精致的浏览器，名为Netscape Navigator，尽管它越来越受欢迎，但缺乏动态的能力。

1995 年为了解决这个问题 Netscape 创建了一种名为 JavaScript 的新脚本语言，该名称纯粹是出于营销目的，因为 Java 是当时热门的新语言

在 1995 年的同一时间，微软推出了他们的浏览器 Internet Explorer，这引发了一场浏览器战争 与 Netscape 一起，微软很快意识到 JavaScript 从根本上改变了Web 的用户体验，并希望Internet Explorer 也能做到这一点，但没有任何规范可供他们遵循，以便与 Netscape 的 JavaScript 相匹配。

1996年，微软对Navigator 解释器进行了反向工程，以创建自己的脚本 称为 jscript 的语言jscript 与 JavaScript 具有相同的用例，但其实现不同，不幸的是，这产生了一个问题，这些差异使开发人员很难使他们的网站在两种浏览器中正常运行。

1996 年 11 月，Netscape 向 ecma International 提交了 JavaScript。

## 什么是 ecma International

它是一个致力于标准化的行业协会。

现在对于每个新规范 ecma 都提供了一个标准规范和一个委员会（对于 JavaScript），该标准称为 Eggman 262 和负责 ik1262 的委员会被称为技术委员会 39 或简称tecma 的 tc39，但决定使用术语ecmascript 来谈论官方语言，原因是收购 Microsystems 的 Oracle 拥有术语 JavaScript 的商标，因此 ecmascript 指的是 标准语言，而 JavaScript 是我们在实践中使用的，并且构建在 ecmascript 之上，多年来我们发布了许多版本的 ecmascript，如您所见，es2015 也称为 es6 是提供我们所谓的现代JavaScript 功能的版本， 已经成为使用 JavaScript 进行开发的先决条件。
